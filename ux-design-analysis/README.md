# UX Design Analysis 技能

> 将产品 PRD 自动拆解为设计师可用的交互文档，内置需求反思环节，输出单个可编辑 HTML 文件。

---

## 适用场景

- 收到产品 PRD（PDF/图片/文字），需要快速输出交互文档
- 想在出图前对需求做一轮 UX 视角的反思和建议
- 需要一份结构化、可编辑、可导出的交互稿依据

---

## 使用方法

### 第一步：提供 PRD

在对话中发送 PRD 文件（支持 PDF、图片、文字），然后说：

```
/ux-design-analysis，分析这个需求
```

### 第二步：回答确认问题（最多 3 个）

AI 会读取 PRD 全部内容（含原型图），筛选出最关键的问题向你确认。

### 第三步：打开 HTML 文件

生成的 HTML 文件包含两个区域：

1. **反思确认区**（初始显示）
   - 3 个需求反思问题：业务目标 → 方案可行性 → UX 风险
   - 每个问题附带建议，你可以选择「同意并修改」或「忽略」
   - 支持补充自己的建议

2. **交互文档区**（确认后显示）
   - 根据你的选择动态生成
   - 采纳的建议会以绿色高亮块标注在对应位置
   - 支持在线编辑、撤销重做、保存、导出 PDF/Markdown

生成完成后会自动在 Finder 中定位该 HTML 文件。

---

## 文件说明

| 文件 | 说明 |
|------|------|
| `SKILL.md` | 技能定义（处理流程、模板结构、样式规范、控件规则） |
| `README.md` | 本文件 |

---

## 核心规则

| 规则 | 说明 |
|------|------|
| 零杜撰 | 所有内容来自 PRD/原型图/用户确认，不足处标 `[待确认]` |
| 面向设计师 | 去技术细节，保留 UI 结构/交互/状态/文案 |
| AB 实验只画实验组 | 对照组是线上版本，无需设计 |
| 最多问 3 个问题 | 只问最关键的，其余标待确认 |
| 反思先行 | 先输出需求反思和建议，确认后再生成文档 |
| 输出即打开 | HTML 落盘后自动在 Finder 中定位 |

---

## 输出格式

单个 HTML 文件：`交互文档_[功能名称].html`

文档结构：01 背景与目标 · 02 用户场景 · 03 交互逻辑 · 04 状态说明 ·（有边界场景时）05 边界场景
