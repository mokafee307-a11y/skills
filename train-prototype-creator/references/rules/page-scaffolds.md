# Page Scaffolds

本文件保存核心火车票页面的稳定基座。生成 `home / list / middle / fill / occupy` 时按需读取。

## 通用执行顺序

```text
业务目标 / 非目标判断 → 选页面基座 → 装配锁定组件模板 → 注入页面数据 → 在允许区域补充自由内容 → 导出预览
```

若用户只要求改某一块组件或补充某类信息，优先在既有槽位处理，不要重写整页。

复用模板前先判断：这个页面块是在帮助用户完成本轮目标，还是只是模板示例里的历史业务语义。若是后者，隐藏、替换或改为更贴合本轮目标的轻量内容。

## 常规买票预订核心路径

完整买票预订流程默认按以下页面串联：

```text
home 首页 → list 列表页 → middle 中间页 → fill 填写页 → occupy 占座页
```

约束：

- `home` 负责搜索入口和预订意图发起。
- `list` 负责车次结果承接、筛选和车次选择。
- `middle` 负责座席 / 预订方案 / sell-card 选择，是常规购票链路的一环。
- `fill` 负责乘车人、联系人、选座、保险等填写确认。
- `occupy` 负责占座成功、待支付票面和支付入口。
- 完整流程、状态跳转和最终交互演示默认不得从 `list` 直接跳到 `fill`，除非用户明确要求简化，或该业务场景没有座席 / 预订方案选择。

## home 首页

默认顺序：

```text
头部 banner → 查询卡片 → 首页二屏内容 → 底部 tab
```

锁定 / 模板优先部分：

- 头部 banner
- 查询卡片
- 吸附在底部的 tab

自由发挥区域：

- 首页二屏内容区，可承载业务主题、营销信息、快捷入口、推荐内容。

约束：

- 优先命中 `assets/templates/home/manifest.json`。
- 即使二屏内容变化，也不得破坏 `banner + 查询卡片 + 底部 tab` 的稳定结构。
- 首页查询卡是订前主决策位。若需求目标是价格利益、积分抵扣、优惠感知、票量提升或预订转化，优先把核心利益点放进查询卡、筛选项、按钮邻近提示或列表承接，不默认新增二屏大卡片。
- 首页查询卡的新增利益点必须是增量表达，不替换基础搜索能力。保留 `有票车次充足`、`优先查看有票方案`、`学生票`、`高铁动车`、`单程/往返/多程` 等模板基础槽位；优惠、积分、权益类信息优先用右上角 tag、按钮邻近轻提示或列表承接表达。
- 首页二屏只作为补充教育位或弱入口。只有当用户需要理解账户权益、规则说明、活动内容或首屏决策位无法承载时，才新增二屏卡片。
- 若本轮不是通勤、常买、收藏或快捷购票场景，二屏内容不得出现车次偏好、乘车人设置、常买车次、快捷预订等旧语义；应改为本轮业务的轻量权益卡、工具入口或直接不展示。

## list 车次列表页

默认顺序：

```text
title bar → 日历组件 → 头部 tab → 车次快筛选项 → 车次卡片 → 底部筛选 tab
```

锁定 / 模板优先部分：

- title bar
- 日历组件
- 头部 tab
- 车次快筛选项
- 车次卡片骨架结构
- 底部筛选 tab

约束：

- 优先命中 `assets/templates/list/manifest.json`。
- 不得因单次需求变化重排六段结构顺序。
- 页面差异优先通过卡片数据、tag、状态、筛选项配置表达。
- 车次卡片内部可在不破坏骨架前提下增补说明文案、tag、状态提示、推荐理由、换乘信息。

## middle 中间页 / 座席服务选购页

默认顺序：

```text
状态栏 → middle-titlebar → middle-train → middle-zuoxi → sell-card 列表 → 底部信任条
```

锁定 / 模板优先部分：

- `middle-titlebar`：返回、日期下拉、须知、分享。
- `middle-train`：出发/到达时刻站点、中部历时、车次与经停、证件/积分/车型说明。
- `middle-zuoxi`：横向座席 Tab，选中态为蓝色描边、蓝字、右下角勾选标识。

灵活区：

- `sell-card` 列表，按 `references/rules/middle-sell-card.md` 注入不同价格、权益、补贴与按钮状态。
- 底部信任条文案可按业务场景调整。

约束：

- 优先命中 `assets/templates/middle/manifest.json`。
- 不得因单次需求变化打散 `middle-titlebar + middle-train + middle-zuoxi` 的顺序。
- Figma 中前缀为 `@svg` 的编组必须以 SVG 资产调用，不得用 CSS 或字符重画。
- `sell-card-*` 的色彩、字体、间距、圆角、组件风格和交互状态必须遵守 `middle-sell-card.md`。

## fill 买票填写页

默认顺序：

```text
title bar → 头部车次信息区 → 乘客组件 → 选座组件 → 贵宾厅组件 → 保险组件 → 底部支付 bar
```

锁定 / 模板优先部分：

- title bar
- 乘客组件
- 选座组件
- 贵宾厅组件
- 保险组件
- 底部支付 bar

灵活区：

- 头部车次信息区是受约束的灵活区，原则为 `屏效优先`、`关键信息披露优先`。

约束：

- 优先命中 `assets/templates/fill/manifest.json`。
- 头部车次信息区右上角必须保留详情下拉入口。
- 可调整车次摘要、时间 / 站点、状态提示、行程摘要等编排。
- `favorite` / 常买车次 / 收藏车次类槽位只在用户明确要求常用、收藏、复用车次时展示。若本轮目标是积分、优惠、保险、选座、乘客填写等其他主题，应隐藏该槽位或替换为对应主题的明细入口。

## occupy 占座页

默认顺序：

```text
头部背景 → title bar → 订单状态 → 票面卡片 → 取消按钮 → 底部支付
```

锁定 / 模板优先部分：

- title bar
- 订单状态
- 票面卡片
- 取消按钮
- 底部支付

约束：

- 优先命中 `assets/templates/occupy/manifest.json`。
- 头部背景属于页面骨架，应与占座状态表达保持一致。
- 额外内容默认只在票面卡片下方扩展，不得挤占、打散或替换订单状态区与票面卡片区。
