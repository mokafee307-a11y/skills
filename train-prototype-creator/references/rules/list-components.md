# List Page Component Rules

本文件保存火车列表页的组件层硬约束。生成列表页、车次卡片或同类组件时按需读取。

## 1. 日历组件

日历组件是锁定组件，结构、选中态、右侧工具入口和露出关系默认按参考稿 / 模板复用，不重新发明日期条样式。

约束：

- 顶部分割线、底部分割线默认不存在，不额外加 `1px` 描边。
- 最下层背景默认透明，不给整条日期轨道再铺额外底色。
- 未选中日期只保留文字与空间占位，不补“每个日期一张小底卡”。
- 选中日期允许保留参考稿中的蓝底高亮块；未选中项不得共享同样块级背景。
- 日历单元严格两行，不新增第三行。
- 普通日期：第一行 `星期几 / 今天`，第二行 `日期`。
- 节日日期：第一行只展示 `list-calendar-vacation.svg`，第二行展示节日名。
- 禁止追加 `推荐`、`低价`、`节前`、`余票稳` 等第三行补充文案。
- 右侧日历工具入口也是两行：第一行日历图标，第二行下拉箭头；不要附加“日历”文字。
- 右侧日历工具入口必须是独立列，末尾露出与遮罩关系稳定，不得让日期文字或图标压到工具入口。

## 2. 车次卡片共用骨架

列表页至少维护两种卡片模板：`直达` 与 `中转`。两类卡片共享同一套横向对齐骨架，不能各自按内容长度自由漂移。

四个固定锚点：

- 出发站区域左对齐。
- 耗时区域居中对齐。
- 到达站区域右对齐。
- 价格区域最右对齐。

混排时，直达 / 中转卡片的出发站、耗时、到达站、价格四个锚点必须跨卡片共线。

## 3. 坐席行

- 坐席行绑定共用骨架，不得单独漂移。
- 第一个坐席块与出发站列左对齐。
- 第二个坐席块与耗时列居中对齐。
- 第三个坐席块与到达站列右对齐。
- 坐席状态收敛为二元语义：`有票` / `无票`。
- `有票` 使用 positive 语义色。
- `无票` 使用 warning / orange 语义色。
- 对外展示时，不可售、不可买、不足量、售罄、停售、已发车、少量等统一渲染为 `无票`，不要混用多个无票语义词。

## 4. 售价块

- 直达 / 中转卡片都必须展示售价块，位置固定在最右侧。
- 售价块默认三层：货币符号、主价格、`起` 后缀。
- 若存在折扣信息，放在主价格下方，用灰色小字。
- 即使是锚定车次、状态车次或说明性车次，也不省略价格列；价格列是共用骨架的一部分。

## 5. Tag

- tag 统一使用同一套 `pill` 形态语言。
- 不得一张卡片用贴片、一张卡片用图片、一张卡片用描边块。
- tag 默认位于卡片左上角；多枚 tag 可横向排列或换行，但仍从左上角起始。
- tag 不得破坏四锚点骨架；拥挤时优先增加卡片高度，不挤压出发 / 耗时 / 到达 / 价格区域。
- 默认语义色：强提示 / 状态提醒 = orange pill；效率 / 推荐 = blue pill；说明性补充 = gray pill。
- 若参考稿中 tag 被锁定，以参考稿为准；若用户要求统一 tag 样式，则在不破坏骨架前提下统一成共享 pill 样式。
